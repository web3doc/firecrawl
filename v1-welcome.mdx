Firecrawl V1版来了！我们引入了更可靠且对开发者友好的API。

以下是新功能：

- `/scrape` 输出格式。选择您希望输出的格式。
- 新的 [`/map` 端点](/features/map) 用于获取网页的大部分URL。
- 为 `/crawl/{id}` 状态提供开发者友好的API。
- 所有计划的速率限制提高了2倍。
- [Go SDK](/sdks/go) 和 [Rust SDK](/sdks/rust)
- 团队支持
- API密钥管理在仪表盘中进行。
- `onlyMainContent` 现在默认为 `true`。
- `/crawl` webhook和websocket支持。


## 抓取格式

现在您可以选择所需的输出格式。可以指定多个输出格式。支持的格式包括：

- Markdown（markdown）
- HTML（html）
- 原始HTML（rawHtml）（无修改）
- 截图（screenshot或screenshot@fullPage）
- 链接（links）
- 提取（extract） - 结构化输出

## 新的抓取Webhook

现在可以在`/crawl`端点上传递`webhook`参数。这将在抓取开始、更新和完成时向您指定的URL发送POST请求。

该webhook现在将为每个抓取的页面触发，而不仅仅是整个结果的最后。

<CrawlWebhookCURL />

### Webhook事件

现在有4种类型的事件：

- `crawl.started` - 当抓取开始时触发。
- `crawl.page` - 为每个抓取的页面触发。
- `crawl.completed` - 当抓取完成时触发，以告知已完成。
- `crawl.failed` - 当抓取失败时触发。


### Webhook响应

- `success` - 如果webhook成功抓取页面。
- `type` - 发生的事件类型。
- `id` - 抓取的ID。
- `data` - 抓取的数据（数组）。这在`crawl.page`上将非空，如果页面成功抓取，则包含1个项目。响应与`/scrape`端点相同。
- `error` - 如果webhook失败，则包含错误消息。

## /scrape端点

更新后的`/scrape`端点已重新设计，以提高可靠性和使用便利性。新的`/scrape`请求体结构如下：

```json
{
  "url": "<string>",
  "formats": ["markdown", "html", "rawHtml", "links", "screenshot", "json"],
  "includeTags": ["<string>"],
  "excludeTags": ["<string>"],
  "headers": { "<key>": "<value>" },
  "waitFor": 123,
  "timeout": 123
}
```

### 格式

现在您可以选择所需的输出格式。可以指定多个输出格式。支持的格式包括：

- Markdown（markdown）
- HTML（html）
- 原始HTML（rawHtml）（无修改）
- 截图（screenshot或screenshot@fullPage）
- 链接（links）
- JSON（json）

默认情况下，输出将仅包括Markdown格式。

### 新请求体的详细信息

下表概述了V1中`/scrape`端点的请求体参数的变化：

| 参数 | 变化 | 描述 |
| --------- | ------ | ----------- |
| `onlyIncludeTags` | 移动并重命名 | 移动到根级别。并重命名为`includeTags`。 |
| `removeTags` | 移动并重命名 | 移动到根级别。并重命名为`excludeTags`。 |
| `onlyMainContent`| 移动 | 移动到根级别。默认为`true`。 |
