## 基本URL
所有请求都包含以下基本URL：
```bash
https://api.firecrawl.dev 
```

## 认证
需要包含一个Authorization头来进行认证。该头应包含`Bearer fc-123456789`，其中`fc-123456789`代表您的API密钥。
```bash
Authorization: Bearer fc-123456789
```
​
## 响应代码
Firecrawl使用常规的HTTP状态码来表示请求的结果。
通常，2xx HTTP状态码表示成功，4xx代码表示与用户相关的失败，而5xx代码表示基础设施问题。

| 状态 | 描述 |
| --- | --- |
| 200 | 请求成功。 |
| 400 | 验证参数的正确性。 |
| 401 | 未提供API密钥。 |
| 402 | 需要支付费用。 |
| 404 | 找不到请求的资源。 |
| 429 | 超过了速率限制。 |
| 5xx | 表示Firecrawl服务器错误。 |

请参阅错误代码部分以获取所有潜在API错误的详细解释。

## 速率限制
Firecrawl API设有速率限制，以确保服务的稳定性和可靠性。速率限制适用于所有端点，基于在特定时间框架内进行的请求数量。
当您超过速率限制时，将收到429响应代码。
