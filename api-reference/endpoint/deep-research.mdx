title: '深度研究'
openapi: 'v1-openapi POST /deep-research'
---

深度研究端点支持通过AI进行任何主题的深度研究和分析。只需提供一个研究查询，Firecrawl将自动探索网络、收集相关信息并将发现综合成全面的洞察。

寻找状态端点？请查看[深度研究状态](/api-reference/endpoint/deep-research-get)端点。

### 响应结构

响应包含：

- **activities**: 研究活动的列表，包括：
  - `type`: 活动类型（'搜索', '提取', '分析', '推理', '综合', '思考'）
  - `status`: 状态（'处理中', '完成', '错误'）
  - `message`: 活动/发现的描述
  - `timestamp`: ISO时间戳
  - `depth`: 研究深度级别

- **sources**: 参考URL，包括：
  - `title`: 来源标题
  - `description`: 来源描述
  - `url`: 来源URL
  - `icon`: 来源favicon

- **finalAnalysis**: 全面分析（完成后）
- **status**: 总体状态（'处理中', '已完成', '失败'）
- **currentDepth**: 当前研究深度
- **maxDepth**: 最大研究深度
- **totalUrls**: 分析的URL数量
- **expiresAt**: 结果过期的ISO时间戳

### 限制

1. 最适合有公开信息的主题
2. 研究作业最多限制为10分钟
3. 建议对关键信息进行手动验证
4. Alpha功能 - 方法和输出可能会演变

### 计费

计费基于分析的URL数量：
- 每个URL = 1个积分
- 使用`maxUrls`参数控制使用量
